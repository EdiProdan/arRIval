# arRIval Documentation (Diátaxis)

This is the canonical documentation home for the project.

The structure follows the Diátaxis framework:

- **Tutorials**: learning-oriented, guided paths for first-time contributors
- **How-to guides**: task-oriented procedures for specific outcomes
- **Reference**: factual contracts, APIs, and architecture details
- **Explanation**: rationale, concepts, and design context

## Start here

- New contributor: [`tutorials/first-local-run.md`](tutorials/first-local-run.md)
- Need to run or verify the system: [`how-to/run-full-stack.md`](how-to/run-full-stack.md)
- Need schema/API facts: [`reference/data-schema.md`](reference/data-schema.md)
- Need architecture reasoning: [`explanation/architecture-overview.md`](explanation/architecture-overview.md)
